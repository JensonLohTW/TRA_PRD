## 從 MySQL 到 PostgreSQL 的主要語法轉換

| MySQL 語法 | PostgreSQL 語法 | 說明 |
|:---|:---|:---|
| `BIGINT UNSIGNED NOT NULL AUTO_INCREMENT` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | 主鍵自增 |
| `BIGINT UNSIGNED` | `BIGINT` | PG 不需要 UNSIGNED 修飾 |
| `TINYINT(1)` | `BOOLEAN` | 布林型別 |
| `TINYINT` / `TINYINT UNSIGNED` | `SMALLINT` | 小整數 |
| `DATETIME(6)` | `TIMESTAMP(6)` | 時間戳 |
| `DATETIME` | `TIMESTAMP` | 時間戳 |
| `CURRENT_TIMESTAMP(6)` | `CURRENT_TIMESTAMP` | 預設值 |
| `ON UPDATE CURRENT_TIMESTAMP` | （已移除，依靠觸發器） | 需另外建立觸發器 |
| `JSON` | `JSONB` | PG 偏好 JSONB |
| `DECIMAL(x,0)` | `DECIMAL(x)` | 簡化 |
| `KEY name (cols)` | `INDEX name (cols)` | PG 不支援行內 KEY |
| `UNIQUE KEY name (cols)` | `UNIQUE name (cols)` | PG 語法 |
| `ENGINE=InnoDB` | （移除） | PG 無引擎差異 |
| `DEFAULT CHARSET=utf8mb4` | （移除） | UTF8 在 cluster 層級 |
| `COLLATE=utf8mb4_unicode_ci` | （移除） | 由 database LC_COLLATE 設定 |
| `COMMENT '...'` 行內註解 | （保留，PG 支援行內 COMMENT） | PG 12+ 支援 |
| `USE db;` | `-- schema via search_path` | PG 用 search_path |
| `DELIMITER //` | （移除） | PG 用 $$ 或單引號 |
| `SET FOREIGN_KEY_CHECKS=0` | （移除） | 用 `SET session_replication_role='replica'` 替代 |

## 安裝步驟

### 1. 建立 PostgreSQL 資料庫

```bash
createdb -E UTF8 -T template0 --lc-collate=en_US.UTF-8 --lc-ctype=en_US.UTF-8 tra_welfare_test
# 或在 psql 中：
# CREATE DATABASE tra_welfare_test
#   ENCODING 'UTF8'
#   LC_COLLATE 'en_US.UTF-8'
#   LC_CTYPE 'en_US.UTF-8'
#   TEMPLATE template0;
```

### 2. 依序安裝模組

```bash
cd sql/postgresql
for f in modules/00_database.sql modules/01_sys.sql ... modules/99_verify.sql; do
    psql -d tra_welfare_test -v ON_ERROR_STOP=1 -f "$f"
done
```

或使用驗證腳本：

```bash
PGDATABASE=tra_welfare_test bash sql/postgresql/scripts/verify_sql.sh
```

### 3. 拼接 all-in-one 整合檔

```bash
bash sql/postgresql/scripts/assemble_sql.sh
# 產生：sql/postgresql/all_in_one.sql
# 執行：psql -d tra_welfare_test -f sql/postgresql/all_in_one.sql
```

## 已知限制與待補強項

1. **`ON UPDATE CURRENT_TIMESTAMP` 替代**：PG 不支援行內 `ON UPDATE`，需建立觸發器函數 `set_updated_at()` 與每張表的 BEFORE UPDATE 觸發器。模組 18_triggers.sql 應補充此機制。
2. **儲存程序語法**：模組 17_functions_procedures.sql 仍是 MySQL 語法（`BEGIN ... END //`、`DELIMITER`、`DECLARE`），需改寫為 PL/pgSQL：
   - `DELIMITER //` → `$$` 或單引號
   - `DECLARE v INT;` → `DECLARE v INT;` 在 `DECLARE` 區塊
   - `MODIFIES SQL DATA`、`READS SQL DATA` → `LANGUAGE plpgsql`
   - `OUT p_business_no VARCHAR(100)` → `OUT p_business_no VARCHAR`
3. **JSON 路徑運算子**：MySQL `JSON_EXTRACT(col, '$.x')` 在 PG 應改為 `col->'x'` 或 `col->>'x'`。
4. **全文搜索**：若使用 `FULLTEXT INDEX`，PG 應改用 `tsvector` + `GIN` 索引。
5. **DISTINCT 與 GROUP BY 行為差異**：MySQL 的 `ONLY_FULL_GROUP_BY` 模式與 PG 的 `GROUP BY` 行為一致，但 PG 更嚴格。
6. **`utf8mb4_unicode_ci` 排序**：PG 預設 `en_US.UTF-8` 可能與 MySQL `utf8mb4_unicode_ci` 在某些字元排序上有微小差異，需測試關鍵字串比較。
