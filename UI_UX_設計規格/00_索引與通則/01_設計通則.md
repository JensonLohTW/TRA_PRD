# 設計通則

## 1. 技術基底

- **前端框架**：Next.js 14+（App Router）
- **UI 組件庫**：Shadcn UI（基於 Radix UI + Tailwind CSS）
- **表單處理**：React Hook Form + Zod 驗證
- **狀態管理**：React Query（Server State）+ Zustand（Client State）
- **樣式方案**：Tailwind CSS v3+
- **圖表**：Mermaid（流程圖）+ Recharts（統計圖表）

## 3. 版面柵格系統

- **柵格**：24 欄網格（`grid grid-cols-24`），適用管理後台複雜版面
- **響應式斷點**：

| 別名 | 寬度 | 目標裝置 |
|------|------|----------|
| sm | ≥ 640px | 手機橫屏 |
| md | ≥ 768px | 平板 |
| lg | ≥ 1024px | 桌面 |
| xl | ≥ 1280px | 寬螢幕 |

- **管理端側欄**：預設展開 256px（`w-64`），可收起為圖示模式（`w-16`）
- **職工端**：單欄流式佈局，最大寬度 768px（`max-w-3xl mx-auto`）

## 4. 表單設計規範

- **標籤**：統一在輸入框上方（`flex-col`），必填欄位標紅色 `*`
- **驗證時機**：輸入即時驗證（`onBlur`）+ 送審前總體驗證
- **錯誤訊息**：紅色文字顯示在欄位下方
- **載入狀態**：提交按鈕顯示 `Loading` spinner，按鈕 disabled
- **樂觀鎖**：所有編輯操作攜帶 row_version，衝突時顯示 Toast 提示「資料已更新，請重新整理」

## 5. 操作反饋規範

| 情境 | 元件 | 持續時間 | 位置 |
|------|------|----------|------|
| 操作成功 | Sonner Toast | 3s | 右上角 |
| 操作失敗 | Sonner Toast + Alert | 5s | 右上角 + 表單頂部 |
| 資料更新衝突 | AlertDialog | 手動關閉 | 置中彈窗 |
| 載入中 | Skeleton / Spinner | 直到完成 | 原地取代 |
| 空狀態 | 插圖 + 說明文字 | 持續 | 列表區域居中 |
