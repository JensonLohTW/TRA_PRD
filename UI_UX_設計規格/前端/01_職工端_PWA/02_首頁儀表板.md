# 首頁儀表板

## 1. 功能概述

職工登入後的第一個頁面，提供快速入口、待辦提醒、近期公告與補助統計摘要。以資訊卡片方式聚合展示。

## 3. 頁面元素與 DB 欄位對應

| UI 元素 | 組件類型 | API/DB 對應 |
|---------|----------|-------------|
| 使用者姓名 | Text | employee.full_name |
| 所屬單位 | Text | org_node.node_name |
| 快速入口卡片 (×4) | Card | 固定路由 |
| 待辦提醒列表 | Card | GET /api/applications?status=returned + GET /api/acknowledgements?status=pending |
| 補助統計 | StatCard | GET /api/applications/statistics?year=current |
| 最新公告列表 | Card | GET /api/announcements?limit=5&status=published |
| 通知圖示 | Button + Badge | GET /api/notifications/unread-count |

## 4. Shadcn UI 組件建議

| 組件 | 用途 | 備註 |
|------|------|------|
| `<Card>` | 快速入口卡片 | selectable, hover 效果 |
| `<Card>` | 待辦提醒/公告 | 附圖示 |
| `<Badge>` | 通知未讀數量 | variant="destructive" |
| `<StatCard>` (自訂) | 統計數字 | 點擊可篩選 |
| `<Skeleton>` | 載入骨架 | 各區塊獨立載入 |
| `<Carousel>` | 公告輪播 (選用) | 首頁頂部輪播 |

## 5. 業務流程圖

```mermaid
flowchart TD
    A[登入成功] --> B[讀取使用者資訊]
    B --> C[並行載入]
    C --> C1[統計資料]
    C --> C2[待辦提醒]
    C --> C3[最新公告]
    C --> C4[未讀通知數]
    C1 & C2 & C3 & C4 --> D[渲染儀表板]

    D --> E1[點擊「我要申請」]
    D --> E2[點擊「我的申請」]
    D --> E3[點擊領款提醒]
    D --> E4[點擊公告]

    E1 --> /benefits
    E2 --> /my-applications
    E3 --> /my-applications/[id]/acknowledge
    E4 --> /announcements/[id]
```

## 6. 互動與狀態

| 狀態 | 處理方式 |
|------|----------|
| Loading | 各區塊獨立 Skeleton 載入 |
| Empty - 無待辦 | 待辦區隱藏或顯示「目前無待辦事項」 |
| Empty - 無公告 | 公告區顯示「暫無最新公告」 |
| Error | 個別區塊顯示重試按鈕，不影響其他區塊 |
