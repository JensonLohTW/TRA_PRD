# 通知中心

## 1. 功能概述

職工檢視所有站內通知，包含系統通知、審核進度通知、公告通知等。支援未讀/已讀分類、單筆標記已讀與全部已讀操作。

## 2. 頁面架構

```
+------------------------------------------+
|  通知中心                                 |
+------------------------------------------+
|  [全部] [未讀] [已讀]                     |
+------------------------------------------+
|  ┌──────────────────────────────────────┐ |
|  │ ● 您的結婚補助已核准 ✅             │ |
|  │    2026/06/20 10:30                  │ |
|  │    [查看詳情]                        │ |
|  ├──────────────────────────────────────┤ |
|  │ ● 子女教育補助申請退回 🔴           │ |
|  │    2026/06/19 14:00                  │ |
|  │    原因：診斷證明模糊                 │ |
|  │    [查看詳情]    [修改重送]          │ |
|  ├──────────────────────────────────────┤ |
|  │ ○ 春節禮金發放通知                  │ |
|  │    2026/06/18 09:00                  │ |
|  │    [查看詳情]                        │ |
|  └──────────────────────────────────────┘ |
|                                          |
|  [全部標記為已讀]                         |
+------------------------------------------+
```

## 3. 頁面元素與 DB 欄位對應

| UI 元素 | 組件類型 | API/DB 對應 |
|---------|----------|-------------|
| 未讀/已讀篩選 | Tabs | notification.is_read |
| 通知卡片 | Card | notification.title, content |
| 未讀標記 | Badge (dot) | notification.is_read = false |
| 通知類型圖示 | Icon | notification.notification_type |
| 通知時間 | Text | notification.created_at |
| 查看詳情 Button | Button | 導向 business_type/business_id |
| 全部已讀 Button | Button | PUT /notifications/read-all |

## 4. Shadcn UI 組件建議

| 組件 | 用途 | 備註 |
|------|------|------|
| `<Tabs>` | 未讀/已讀篩選 | 全部/未讀/已讀 |
| `<Card>` | 通知列表項 | 點選標記已讀 |
| `<Badge>` | 未讀紅點 | variant="destructive" (dot) |
| `<ScrollArea>` | 長列表滾動 | 固定高度 |
| `<Button>` | 全部已讀 | variant="ghost" |
| `<Separator>` | 列表分隔 | - |

## 5. 通知類型與圖示映射

| notification_type | 圖示 | 色彩 |
|-------------------|------|------|
| workflow | ClipboardCheck | primary |
| alert | AlertTriangle | destructive |
| system | Bell | info |
| announcement | Megaphone | accent |

## 6. 互動與狀態

| 狀態 | 處理方式 |
|------|----------|
| Loading | Skeleton × 5 |
| Empty - 無通知 | 插圖 +「目前無通知」 |
| Empty - 無未讀 | 「所有通知已讀取」 |
| Click | 單擊標記已讀 + 導向業務頁面 |
| Edge - 業務資料已刪除 | 仍保留通知文字，導向時顯示「此案件已不存在」 |

## 7. 相關頁面與路由

- 通知中心：/notifications
- 點選通知 → 依 business_type 導向對應頁面（申請詳情/公告詳情等）
