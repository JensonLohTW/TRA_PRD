# 領款與異議管理

## 1. 功能概述

管理端監控領款確認狀態，查詢已確認與未確認案件，以及處理職工提出的領款異議案件。

## 2. 頁面架構

### 領款確認管理（/admin/payment/acknowledgements）

```
+------------------------------------------+
|  領款確認管理                     [匯出] |
+------------------------------------------+
|  [待確認 3] [已確認 25] [已異議 2]        |
+------------------------------------------+
|  ┌────┬──────┬──────┬──────┬──────┬──┐  |
|  │案號 │申請人│金額  │撥款日│確認狀態│操作│  |
|  ├────┼──────┼──────┼──────┼──────┼──┤  |
|  │TP.. │王小明│12,000│06/25│待確認 │[提醒]│
|  │TP.. │李四  │8,000 │06/20│已確認 │[查看]│
|  │TP.. │陳五  │5,000 │06/20│已異議 │[處理]│
|  └────┴──────┴──────┴──────┴──────┴──┘  |
+------------------------------------------+
```

## 3. 頁面元素與 DB 欄位對應

| UI 元素 | 組件類型 | API/DB 對應 |
|---------|----------|-------------|
| 狀態統計 | StatCard | payment_acknowledgement.status |
| 確認列表 | DataTable | payment_acknowledgement + payment_batch_item |
| 提醒 Button | Button | POST /notifications/send (手動催辦) |
| 異議資訊 | Card | payment_dispute_case |
| 異議原因 | Text (readonly) | payment_dispute_case.dispute_reason |
| 處理說明 Textarea | Textarea | payment_dispute_case.latest_comment |
| 異議成立 Button | Button | POST /payment/disputes/{id}/resolve |
| 異議不成立 Button | Button | POST /payment/disputes/{id}/reject |
| 結案 Button | Button | POST /payment/disputes/{id}/close |

## 4. Shadcn UI 組件建議

| 組件 | 用途 | 備註 |
|------|------|------|
| `<StatCard>` (自訂) | 統計 | 待確認/已確認/已異議 |
| `<DataTable>` (自訂) | 列表 | - |
| `<Card>` | 案件/異議資訊 | - |
| `<Textarea>` | 處理說明 | - |
| `<Button>` | 操作按鈕 | variant 區分 |
| `<ConfirmDialog>` (自訂) | 異議處理確認 | - |

## 5. 相關頁面與路由

- 領款確認列表：/admin/payment/acknowledgements
- 異議案件列表：/admin/payment/disputes
- 異議案件處理：/admin/payment/disputes/[id]
